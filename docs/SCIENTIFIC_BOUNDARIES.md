# Scientific Boundaries

## Purpose
This project demonstrates environmental engineering workflows in software form and is intended for portfolio and prototype use.

## Data Source Scope
- Current dashboard telemetry is generated by a local synthetic data module for deterministic demos.
- Measurements are not directly connected to certified FRM/FEM field instruments in this repository state.

## Modeling Scope
- `aermod_simulator.py` and `calpuff_simulator.py` are simplified representations for visualization and educational interpretation.
- They are not configured as a complete regulatory dispersion workflow (no full terrain/land-use stack, formal met preprocessing, or authority-approved modeling protocol).

## Recommended Upgrade Path
- Integrate validated instrument pipelines with calibration/QA metadata.
- Version-control assumptions for meteorology and emissions inventory.
- Add uncertainty reporting and model verification against benchmark scenarios.
